# WPAI Themes

**Free, open-source WordPress themes & plugins — built in public, previewable in your browser, installable in one click.**

Every theme and plugin in this repo is 100% free under the GPL. Browse the live gallery, preview any theme running *real WordPress in your browser* (no install), and download a ready-to-upload `.zip`.

🔗 **Live gallery:** https://lordbasilaiassistant-sudo.github.io/wpai-themes/

---

## What's in here

```
wpai-themes/
├── themes/            # One folder per WordPress theme (classic, GPL-licensed)
│   └── aurora/        #   style.css + PHP templates + screenshot.svg
├── plugins/           # One folder per WordPress plugin
│   └── reading-time-badge/
├── docs/              # The GitHub Pages gallery (static; live-previews via WP Playground)
├── scripts/           # Build tooling — zips, screenshots, gallery manifest, local dev
└── .github/workflows/ # CI: rebuilds the gallery and deploys Pages on every push
```

## For users — get a theme

1. Open the [live gallery](https://lordbasilaiassistant-sudo.github.io/wpai-themes/).
2. Click **Live Preview** to run the theme in a throwaway WordPress in your browser.
3. Click **Download** to grab the `.zip`.
4. In your site: **Appearance → Themes → Add New → Upload Theme**, pick the zip, **Activate**.

Plugins install the same way under **Plugins → Add New → Upload Plugin**.

## For developers — build & test locally

No Docker, no MySQL, no XAMPP. Local WordPress runs on PHP-WASM via [`@wordpress/wp-now`](https://www.npmjs.com/package/@wordpress/wp-now).

```bash
npm install                  # one-time: build tooling (archiver + sharp)

npm run dev aurora           # boot real WordPress with the "aurora" theme active
                             #   → http://localhost:8881  (admin: admin / password)

npm run build                # generate screenshots, zips, and gallery.json into docs/
npm run preview              # build, then serve the gallery at http://localhost:4000
```

`npm run dev <name>` mounts `themes/<name>/` (or `plugins/<name>/`) into a fresh WordPress
and activates it. Edit files, refresh the browser, see changes instantly.

## How it works

- **Themes are the source of truth.** The gallery is generated by parsing each theme's
  standard WordPress `style.css` header (and each plugin's main-file header) — no separate
  catalog to maintain. Add a folder, push, and it shows up.
- **Live previews** use [WordPress Playground](https://wordpress.github.io/wordpress-playground/):
  the gallery hands Playground a *blueprint* that installs the theme's zip and activates it,
  all in a browser sandbox.
- **CI** (`.github/workflows/deploy.yml`) runs `npm run build` and publishes `docs/` to
  GitHub Pages on every push to `main`.
